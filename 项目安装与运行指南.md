## 1. 安装 Bun

本项目使用 **Bun** 作为包管理器和运行环境。项目要求 **Bun 1.3+**（`package.json` 中声明 `"packageManager": "bun@1.3.13"`）。

### 1.1 下载 Bun

打开 **PowerShell**，执行：

```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
```

脚本会自动下载 Bun 并添加到系统 PATH。


### 1.2 验证安装

```powershell
bun --version
```

输出示例：

```
1.3.13
```

> 如果提示 `bun 不是可识别的命令`，请**重启终端**或**重新加载用户环境变量**。

## 2. 安装项目依赖

在项目根目录执行：

```powershell
bun install
```

这会：

- 安装根目录依赖
- 解析 workspace 中各子包（`packages/*`、`packages/console/*` 等）的依赖
- 自动触发 `postinstall` 脚本（执行 `fix-node-pty`）

### ❗可能遇到的问题

#### 2.1 Electron 下载失败

`packages/desktop` 依赖 `"electron": "42"`。`bun install` 期间，electron 包会自动下载其二进制文件（`electron-v42.0.0-win32-x64.zip`）并解压到 `node_modules/electron/dist/`。

但由于网络原因，大概率会**下载失败或超时**。这时你会看到类似：

```
Downloading electron-v42.0.0-win32-x64.zip
Error: Client network socket disconnected before secure TLS connection was established
```

**这是正常现象**，不影响 `bun install` 本身的完成。请继续到下一步手动处理。

#### 2.2 验证依赖安装

```powershell
bun ls
```

确认 `packages/desktop` 下的 electron 包已出现：

```
└─ electron@42.0.0
```

如果 electron 不存在，可单独安装：

```powershell
cd packages\desktop
bun add electron@42
cd ..\..
```

## 3. 处理 Electron 二进制文件

这是最关键的一步。需要手动下载 Electron v42.0.0 的 Windows 二进制包并放置到正确位置。

### 3.1 下载 Electron zip

**下载地址**（使用镜像以加速）：

```
https://github.com/electron/electron/releases/download/v42.0.0/electron-v42.0.0-win32-x64.zip
```

将下载好的 `electron-v42.0.0-win32-x64.zip` 放入项目根目录（`d:\60096960\UXAI-dev_pattern\`）。

### 3.2 解压到正确位置

```powershell
# 创建 dist 目录
mkdir -Force packages\desktop\node_modules\electron\dist

# 解压 zip 到 dist 目录
Expand-Archive -Path "electron-v42.0.0-win32-x64.zip" -DestinationPath "packages/desktop/node_modules/electron/dist" -Force
```

### 3.3 创建安装标记文件

Electron 包的 `install.js` 通过检测是否存在 `path.txt` 和 `dist/version` 来判断是否已安装。需要手动创建：

```powershell
Set-Content -Path "packages/desktop/node_modules/electron/path.txt" -Value "electron.exe" -NoNewline
```

### 3.4 验证安装

确认以下文件均已存在：

```powershell
# 检查 electron.exe
Get-ChildItem packages\desktop\node_modules\electron\dist\electron.exe

# 检查版本文件
Get-Content packages\desktop\node_modules\electron\dist\version

# 检查标记文件
Get-Content packages\desktop\node_modules\electron\path.txt
```

预期输出：

- `electron.exe` 存在（约 226MB）
- `version` 文件内容：`42.0.0`
- `path.txt` 文件内容：`electron.exe`

## 4. 运行开发服务

### 4.1 首次运行

在项目根目录执行：

```powershell
bun run dev:desktop
```

该命令会依次：

1. **执行 `predev` 脚本** — 复制图标、编译 `packages/opencode` 的 Node 端代码（生成 `skills.json`、部署内置技能、原型等）
2. **启动 `electron-vite dev`** — 使用 Vite 构建 Electron 的 main/preload/renderer 并启动 Electron 窗口

### 4.2 启动后的现象

正常启动时，终端会输出类似：

```
$ bun run --cwd packages/desktop dev
$ bun ./scripts/predev.ts

$ electron-vite dev
vite v7.1.4 building SSR bundle for development...
...
[octo:env] ┌─ effective env  mode=  channel=dev  command=serve  (...)

app starting { version: '1.14.41', packaged: false, ... }
DevTools listening on ws://127.0.0.1:9222/...
sidecar connection started { url: 'http://127.0.0.1:60509' }
```

同时会弹出 Electron 应用窗口。
